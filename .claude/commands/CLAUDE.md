# .claude/commands/
> L2 | 父级: /CLAUDE.md

Skills 入口 — 13个 /命令，通过 Claude Code 的 `/命令名` 调用。

## 成员清单

### 创作类
`母稿.md`: 核心长文创作引擎（800-1500字），所有平台内容的源头。读取 config/creator.yaml 执行三轮润色 + 质量门控
`小红书.md`: 小红书图文（500-800字）+ 封面生成 + 卡片渲染 + 可选发布。最复杂的 skill，含完整后处理链
`公众号.md`: A组长图文，一次生成公众号/百家号/今日头条三个版本（正文相同，标题各异）
`即刻.md`: 即刻短文本（100-300字），润色力度最大，最口语化
`X推文.md`: 英文推文（280字符），英文思维直接创作非翻译，三轮英文润色
`linuxdo.md`: 技术社区帖（零营销感），含冷启动期检查和营销感检测
`GitHub.md`: Discussion/PR/Issue 草稿，英文为主，含周节奏提醒
`朋友圈.md`: 私域内容，按星期模板（产品/观察/趣味/技术/互动/日常）

### 工具类
`合规检查.md`: 内容合规审查，被所有创作类 skill 内部调用。读取 config/compliance.yaml
`卡片渲染.md`: 独立调用卡片渲染引擎（8主题 × 4分页模式）
`全平台分发.md`: 一键从母稿衍生全平台内容，内联执行所有平台逻辑，含频率控制和冷启动检查

### 管理类
`今日任务.md`: 根据 Day N 和日类型生成待办清单，读取 config/schedule.yaml
`周复盘.md`: 扫描 output/ 产出统计，生成周复盘报告

## 调用链

```
/今日任务 → 查看今日待办
/母稿 → 创作核心长文
    ├→ /全平台分发 → 一键衍生（内联调用以下全部）
    │   ├→ 公众号逻辑（+ 百家号 + 今日头条）
    │   ├→ 小红书逻辑（+ 封面 + 卡片）
    │   ├→ 即刻逻辑
    │   ├→ X推文逻辑
    │   ├→ linuxdo逻辑（频率控制）
    │   ├→ 朋友圈逻辑
    │   └→ GitHub逻辑（可选）
    │
    └→ 或单平台调用：/小红书 /公众号 /即刻 ...

/合规检查 ← 被所有创作类 skill 内部调用
/周复盘 → 周日使用
```
