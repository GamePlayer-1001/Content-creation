## 10 种改写风格（A-J）

### A — 个人故事型
- **模板**：`templates/洗稿-个人故事型.md`
- **核心**：第一人称叙事，用真实经历展现观点
- **结构**：故事 → 感悟 → 升华
- **写法**：具体时间地点人物（不要"有一天"）、细节描写（对话/心理/场景）、情绪真实不避讳脆弱
- **禁忌**：❌ "在我人生的某个阶段..." ❌ "深深地感动了我" ❌ "这让我明白了一个道理"
- **AI检测风险**：低
- **推荐平台**：知乎、Medium、Reddit

### B — 对话场景型
- **模板**：`templates/洗稿-对话场景型.md`
- **核心**：场景+对话重现，画面感强
- **结构**：场景+人物 → 对话展开 → 思考/行动
- **写法**：典型场景（咖啡馆/办公室）、两人对话、口语化有停顿有情绪、场景细节（环境/动作/表情）
- **对话技巧**：✅ "其实吧""说实话""你懂的" ✅ 有停顿"我...我当时真的..." ❌ 避免"因此""然而""诚然"
- **AI检测风险**：低
- **推荐平台**：公众号、小红书

### C — 观点论述型（新增）
- **核心**：理性分析+案例支撑，逻辑层层递进
- **结构**：核心观点 → 分论点1/2/3 → 案例/数据 → 升华
- **写法**：开门见山提核心观点、3-5个分论点、每点用案例或数据支撑、逻辑递进
- **禁忌**：❌ 过度使用"首先""其次""最后" ❌ 假大空案例 ❌ 空洞论述
- **AI检测风险**：⚠️ 中高（需额外注入瑕疵）
- **推荐平台**：知乎、Medium、Quora

### D — 情感共鸣型
- **模板**：`templates/洗稿-情感共鸣型.md`
- **核心**：强化情绪表达，直击读者内心
- **结构**：触发共鸣 → 描绘痛点 → 分析 → 出路 → 希望
- **写法**：用"你"直接对话、描绘读者痛点心声、共情不煽情
- **情绪技巧**：✅ "我知道那种感觉，真的很难受" ✅ "凌晨两点躺在床上睡不着" ❌ "深深的痛苦""巨大的压力"
- **AI检测风险**：低
- **推荐平台**：公众号、小红书、即刻

### E — 干货清单型（新增）
- **核心**：结构化呈现，要点清晰实用
- **结构**：问题/需求 → 要点1/2/3... → 总结/行动清单
- **写法**：小标题/序号清晰分段、每点独立完整、提供可执行方法、简洁删废话
- **特点**：高收藏率，适合工具推荐/方法论/教程类
- **AI检测风险**：中
- **推荐平台**：知乎、Quora、小红书

### F — 视频口播型（新增）
- **核心**：口语化+节奏感，最难被AI检测 🔥
- **结构**：3秒钩子 → 3-30秒引话题 → 30-90秒核心 → 互动结尾
- **写法**：前3秒强钩子（疑问/冲突/金句）、短句为主、语气词频繁、设置停顿和情绪起伏
- **口语特征**：大量"说真的""我靠""怎么说呢"、碎碎念、思考过程外化
- **AI检测风险**：最低 🔥
- **推荐平台**：即刻、朋友圈、小红书

### G — 技术深度型（新增）
- **核心**：代码示例+原理分析+技术细节+性能对比
- **结构**：问题/需求 → 原理 → 方案 → 代码 → 性能 → 最佳实践
- **写法**：技术原理清晰、代码可运行、对比不同方案、踩坑经验分享
- **⚠️ 代码块保护**：三反引号/单反引号/缩进代码 100% 不可修改
- **AI检测风险**：中低（技术内容天然不太像AI）
- **推荐平台**：linuxdo、知乎、GitHub

### H — 争议讨论型（新增）
- **核心**：反直觉观点+引战+争议性+预设反驳
- **结构**：反常识观点 → 大众认知 → 我的不同意 → 论据 → 邀请讨论
- **写法**：开头抛反直觉观点、充分论据、预设反驳并回应、保持理性不人身攻击
- **特点**：高互动率，适合引发讨论
- **AI检测风险**：低（争议性天然不像AI）
- **推荐平台**：X、Reddit、知乎

### I — 轻松娱乐型（新增）
- **核心**：玩梗+调侃+年轻化表达+适度自嘲
- **结构**：热梗/段子切入 → 轻松展开 → 穿插调侃 → 有趣总结
- **写法**：网络热梗自然融入、调侃不冒犯、年轻化用语、轻松幽默、适度自嘲
- **AI检测风险**：低（梗和自嘲是AI的弱项）
- **推荐平台**：小红书、即刻、Reddit

### J — 专业权威型（新增）
- **核心**：数据引用+学术风格+严谨论证+可读性
- **结构**：背景+价值 → 现状分析 → 深入分析 → 解决方案 → 结论+展望
- **写法**：引用可靠数据源、专业术语恰当、逻辑严谨论证充分、客观中立、保持可读性
- **AI检测风险**：⚠️ 中高（学术风格容易被判AI，需注入个人观点）
- **推荐平台**：Medium、Quora、知乎

## 第三步：改写

### 核心原则
1. 保持原文核心观点和信息不变
2. 彻底改变行文风格、句式结构、叙事角度
3. 应用 creator.yaml 中 anti_detection 策略
4. 遵守 creator.yaml 中 blacklist（中文黑名单零容忍）

### 反 AI 检测操作
从 creator.yaml 的 anti_detection.strategies 逐条执行：
- 打破对仗句式（避免 A,B;A,B 的平行结构）
- 增加口语碎碎念（插入无意义的感叹/自嘲，如"说真的"、"我靠"、"怎么说呢"）
- 减少概括性总结句（删除"总的来说"、"综上所述"）
- 用具体场景替代抽象概念（"效率提升50%" → "原来半天的活，现在午饭前就搞完了"）
- 术语口语化（"吃什么进去的"替代"输入了什么数据"）
- 陈述句 → 思考过程句（"我琢磨过"、"后来我想明白了"）
- 格式化 → 随意化（去掉工整的列表，变成自然段落）
- 逻辑递进 → 故事展开（时间线叙事替代论证）

### 7 种真实瑕疵注入（必须主动添加）
1. **重复强调** — 关键观点重复 2-3 次
2. **情绪失控** — 3-5 处情绪化表达
3. **题外话** — 1-2 处突然联想
4. **模糊数据** — 部分数据用"好像""大概"
5. **自我修正** — 1-2 处"不对，应该说是..."
6. **口语废话** — 5-8 处停顿和过渡词
7. **不均匀段落** — 有的 1 句，有的 10 句

### 三轮改写
1. **结构改写**：按风格模板重组段落顺序、叙事角度
2. **句式改写**：逐句检查，消灭 AI 痕迹（黑名单零容忍）
3. **人味注入**：添加 7 种真实瑕疵 + 口语碎碎念（10-15%比例）

### 代码块保护
如果文章包含代码块：
1. 提取代码块 → 替换为占位符 `{{CODE_BLOCK_N}}`
2. 只对非代码区域应用改写规则
3. 还原代码块，验证未被修改

## 第四步：质量门控

### AI 检测评估（自评）
对改写结果逐项检查：
- [ ] 无连续 3 句以上的对仗/排比
- [ ] 口语碎碎念占比 >= 10%
- [ ] 无 creator.yaml blacklist 中的任何词汇
- [ ] 具体场景数 >= 3 处
- [ ] 有至少 2 处不完美表达（口误/自嘲/犹豫）
- [ ] 短句(≤10字)占比 25-35%
- [ ] 连续长句不超过 5 句
- [ ] 情绪爆点 >= 2 个/千字

评分 < 8/10 → 回到第三步继续打磨

### 合规检查
调用 `/合规检查` 的规则对改写内容审查
