## 执行流程

### 第一步：加载规则库

读取 `D:/Software/内容生成输出/config/compliance.yaml`，加载完整规则库：
- 敏感词黑名单（political / superstition / commercial / prohibited / vulgar）
- 功效暗示红线（forbidden + safe_alternatives）
- 导流行为禁止（anti_diversion）
- 谨慎词及频次限制（caution_words，每条 <= max_per_post 次）
- 内容真实性规则（authenticity）
- 不当行为检测（behavior）

### 第二步：获取待检内容

根据用户输入确定待检内容：
- 如果用户提供了文件路径 → 读取该文件内容
- 如果用户直接粘贴了文本 → 使用粘贴的文本
- 如果没有任何输入 → 提示用户提供内容

### 第三步：逐项扫描

对待检内容执行以下检查（每项独立报告）：

#### 3.1 敏感词黑名单扫描
- 逐词扫描 blacklist 中所有分类的敏感词
- 包括变体检测（谐音、拆字、拼音缩写等常见规避手段）
- 命中任何一个 = 严重违规

#### 3.2 功效暗示模式检测
- 扫描 efficacy_patterns.forbidden 中的所有模式
- 检查是否存在"动作 + 功效结果"的语义组合
- 即使单词不在列表中，语义上构成功效暗示也要标记

#### 3.3 导流行为检测
- 扫描 anti_diversion.forbidden 中的所有项
- 检查是否存在引导用户离开平台的暗示

#### 3.4 谨慎词频次检查
- 统计 caution_words.words 中每个词的出现次数
- 超过 max_per_post 次的标记为警告

#### 3.5 内容真实性检查
- 扫描 authenticity.forbidden_claims 中的声明
- 检查是否存在伪造真实性的表述

#### 3.6 不当行为检测
- 扫描 behavior 中的 unfriendly / fraud / cheating 模式

### 第四步：生成审查报告

输出格式：

---

合规审查报告

审查状态：✅ 通过 / ⚠️ 有风险 / ❌ 不通过

--- 详细结果 ---

1. 敏感词黑名单：✅ 无命中 / ❌ 命中 N 个
   - [列出命中的词及所在句子]

2. 功效暗示：✅ 无命中 / ❌ 命中 N 处
   - [列出命中的表述]
   - 建议替换为：[从 safe_alternatives 中推荐]

3. 导流行为：✅ 无命中 / ❌ 命中 N 处
   - [列出命中的表述]

4. 谨慎词频次：✅ 正常 / ⚠️ 超频
   - [列出超频的词及出现次数]

5. 内容真实性：✅ 正常 / ❌ 存在虚假声明
   - [列出问题表述]

6. 不当行为：✅ 正常 / ❌ 检出问题
   - [列出问题]

--- 修改建议 ---
[针对每个不通过项，给出具体的修改建议和安全替代表述]

---

### 第五步：判定总结

判定规则：
- 任何一项 ❌ = 总体判定 ❌ 不通过
- 只有 ⚠️ 无 ❌ = 总体判定 ⚠️ 有风险（建议修改但可发布）
- 全部 ✅ = 总体判定 ✅ 通过
