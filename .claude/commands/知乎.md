## 第三步：创作/适配

### 语气风格
- 专业 + 深度 + 数据支撑
- 有理有据，可反驳但要礼貌，避免绝对化
- 长度：1500-3000字

### 内容类型（自动判断）
1. **问答型**：针对具体问题深度回答 + 亲身经历 + 逻辑清晰
2. **专栏文章型**：系统性知识输出 + 独到观点 + 行业洞察

### 标题公式
- 深度分析型："[领域]的[数量]个真相，[结果]" / "深度解析：为什么[现象]"
- 反常识型："[普遍认知]可能是错的" / "关于[话题]，你需要知道的[数量]件事"
- 经验分享型："我在[领域]工作[时间]，总结出这些经验"

### 回答结构
- **开头**：资历背书型 / 直接观点型 / 反驳型（三选一）
- **主体**：三段式（3-5个分论点，每点有案例/数据支撑）
- **结尾**：总结型（简洁回扣核心观点）或互动型（抛出讨论问题）

### 知乎必须加入的元素（降AI风险）
1. **亲身经历**：至少1段个人故事
2. **具体细节**：具体的数字、时间、地点
3. **不完美表达**："说实话"、"可能我理解有偏差"
4. **争议观点**：提出可讨论的点

### 知乎高风险写法（必须规避）
| 高风险 | 改进方案 |
|--------|---------|
| 首先...其次...最后 | 用数字标号或小标题 |
| 根据研究表明 | 具体说出哪个研究 |
| 综上所述 | 用"简单来说"或删除 |
| 值得注意的是 | 删除，直接说 |

### 三轮润色
从 creator.yaml polish_engine 加载，知乎润色力度适中：
- 第二轮人类废话比例：8-12%
- 第三轮倒装句：2-3 个
- 比小红书稍正式，但比公众号更直率
