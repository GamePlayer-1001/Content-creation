## 核心组件详解

### 1. 定时调度器（Daily Scheduler）

**文件**：`automation/schedulers/daily_scheduler.py`

**职责**：
- 注册定时任务
- 协调各模块运行
- 错误处理和重试
- 日志记录

**时间表**：
```
06:00 → 抓取热点
07:00 → 生成早间内容
08:00 → 发布早间内容

11:00 → 刷新热点
11:30 → 生成午间内容
12:30 → 发布午间内容

17:00 → 刷新热点
17:30 → 生成晚间内容
19:30 → 发布晚间内容
```

**依赖**：
- `schedule` 库（Python）
- 所有其他模块

---

### 2. 热点抓取器（Trending Scraper）

**文件**：`automation/scrapers/xhs_trending_scraper.py`

**技术栈**：
- Playwright（Python）
- 异步编程（asyncio）

**工作流程**：
```
启动浏览器
    ↓
访问小红书首页
    ↓
提取热门话题
    ↓
搜索12个关键词
    ↓
提取热门标签
    ↓
保存到 JSON
```

**关键词列表**：
- 塔罗、占卜、运势、星座
- 生肖、八字、紫微斗数、命理
- 风水、水晶、冥想、心灵成长

**防封策略**：
- 移动端 User-Agent
- 随机延迟 2-5 秒
- 每分钟最多 10 个请求

---

### 3. 内容管道（Content Pipeline）

**文件**：`automation/generators/content_pipeline.py`

**职责**：
- 协调内容生成和渲染
- 管理文件保存
- 生成元数据

**工作流程**：
```
加载热点数据
    ↓
调用内容生成器
    ↓
保存 Markdown
    ↓
调用图片渲染器
    ↓
保存元数据
    ↓
返回结果
```

**输出文件**：
- `content.md` - Markdown 内容
- `metadata.json` - 元数据
- `cover.png` - 封面图
- `card_*.png` - 卡片图

---

### 4. 内容生成器（Content Generator）

**文件**：`automation/generators/claude_content_generator.py`

**当前模式**：示例内容模式

**时段内容**：
```
Morning（早间）
├─ 风格：轻松、正能量
├─ 主题：每日运势、开运方法
├─ 长度：200-400字
└─ 推广：自然提及

Midday（午间）
├─ 风格：专业、干货向
├─ 主题：塔罗解读、命理知识
├─ 长度：500-800字
└─ 推广：资源分享

Evening（晚间）
├─ 风格：温暖、疗愈向
├─ 主题：情感占卜、心灵成长
├─ 长度：600-900字
└─ 推广：用户故事
```

**输出格式**：
```markdown
---
emoji: "🌟"
title: "标题"
subtitle: "副标题"
---

# 正文内容

## 章节1

内容...

---

💡 推广文案

#标签1 #标签2 #标签3
```

---

### 5. 图片渲染器（Image Renderer）

**文件**：`automation/publishers/image_renderer.py`

**技术栈**：
- Node.js
- Playwright（Node）
- render_xhs_v2.js

**渲染流程**：
```
读取 Markdown
    ↓
解析 YAML frontmatter
    ↓
应用主题样式
    ↓
智能分页
    ↓
截图生成 PNG
    ↓
保存图片文件
```

**支持的主题**：
```
早间：mint, sunset, xiaohongshu
午间：elegant, ocean, purple
晚间：dark, sunset, purple
```

**图片规格**：
- 尺寸：1080×1440 像素
- 格式：PNG
- DPR：2（高清）
- 自动分页：封面 + N 张卡片

---

### 6. 自动发布器（Playwright Publisher）

**文件**：`automation/publishers/playwright_publisher.py`

**技术栈**：
- Playwright（Python）
- 异步编程（asyncio）

**工作流程**：
```
启动浏览器
    ↓
加载 Cookie
    ↓
访问发布页面
    ↓
检测登录状态
    ├─ 未登录 → 手动登录 → 保存 Cookie
    └─ 已登录 → 继续
    ↓
上传图片
    ↓
填写标题
    ↓
填写描述和标签
    ↓
等待用户确认
    ↓
点击发布按钮
    ↓
获取笔记 URL
    ↓
归档内容
```

**Cookie 管理**：
```
首次运行
    ↓
检测无 Cookie
    ↓
打开浏览器
    ↓
提示手动登录
    ↓
保存 Cookie 到 JSON
    ↓
后续运行自动加载
```

## 错误处理

### 错误处理流程

```
任务执行
    ↓
发生错误？
    ├─ 是 → 记录日志
    │       ↓
    │     重试机制
    │       ├─ 重试成功 → 继续
    │       └─ 重试失败 → 保存到失败队列
    │
    └─ 否 → 继续执行
```

### 常见错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| Cookie 过期 | 提示手动登录 |
| 网络超时 | 自动重试 3 次 |
| 渲染失败 | 记录日志，跳过 |
| 发布限流 | 指数退避重试 |
| 文件不存在 | 创建目录后重试 |

## 性能优化

### 并发策略

- 热点抓取：异步并发搜索
- 内容生成：按时段串行
- 图片渲染：单进程（Node.js）
- 自动发布：单进程（避免限流）

### 资源管理

```
内存使用：
├─ Python 进程：~100 MB
├─ Node.js 进程：~200 MB
├─ Chrome 进程：~500 MB
└─ 总计：~800 MB

磁盘使用：
├─ 代码：~5 MB
├─ 依赖：~500 MB
├─ 日志：~10 MB/月
├─ 内容：~50 MB/月
└─ 总计：~565 MB

网络使用：
├─ 热点抓取：~5 MB/次
├─ 图片上传：~300 KB/张
└─ 每日总计：~20 MB
```

## 安全性设计

### Cookie 安全

```
存储方式：
├─ 文件：xhs_cookies.json
├─ 权限：仅当前用户
├─ 加密：无（本地存储）
└─ 有效期：30-90天

建议：
├─ 定期更新
├─ 不要分享
└─ 不要提交到 Git
```

### API 限流保护

```
请求频率：
├─ 热点抓取：每分钟最多 10 次
├─ 自动发布：每天最多 3 次
└─ 延迟策略：随机 2-5 秒

避免行为：
├─ 短时间大量请求
├─ 规律性重复操作
└─ 异常高频访问
```
