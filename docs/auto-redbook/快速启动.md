## 第一步：测试自动发布（必须）

```bash
python test_publish_manual.py
```

### 首次运行流程

1. **脚本显示准备发布的内容信息**
   - 标题：今日运势来啦
   - 图片：3张（cover.png + 2张卡片）
   - 标签：今日运势、星座运程、塔罗占卜等

2. **按回车后，浏览器自动打开**
   - Chrome 浏览器会自动打开
   - 访问小红书创作中心

3. **手动登录小红书（首次）**
   - 如果显示登录页面，请扫码或密码登录
   - 登录成功后，在命令行按回车
   - Cookie 会自动保存到 `automation/config/xhs_cookies.json`

4. **系统自动操作**
   - 自动上传3张图片
   - 自动填写标题
   - 自动填写描述和标签

5. **确认发布**
   - 在浏览器中检查内容是否正确
   - 在命令行按回车确认发布

6. **完成**
   - 显示发布结果和笔记 URL
   - 浏览器5秒后自动关闭

### 成功标志

```
[OK] 发布成功！
笔记 URL: https://www.xiaohongshu.com/explore/xxxxxx
时间: 2026-02-06T13:30:00
```

## 第二步：启动自动化调度器

### 方式 A：临时运行（测试用）

```bash
run_scheduler.bat
```

- 适合测试和短期运行
- 关闭命令行窗口即停止
- 可以看到实时日志输出

### 方式 B：配置为开机启动（推荐）

#### 详细步骤

1. **打开任务计划程序**
   - 按 `Win + R`
   - 输入 `taskschd.msc`
   - 回车

2. **创建新任务**
   - 右侧点击"创建基本任务"

3. **配置任务**
   - **名称**：`DestinyTeller-AutoScheduler`
   - **描述**：Destinyteller 小红书自动化发布系统
   - **触发器**：计算机启动时
   - **操作**：启动程序
   - **程序/脚本**：点击"浏览"，选择
     ```
     h:\Project\Auto-Redbook-Skills\run_scheduler.bat
     ```

4. **高级设置**
   - 取消勾选"只有在计算机使用交流电源时才启动"
   - 勾选"允许按需运行任务"
   - 勾选"如果任务失败，重新启动"（可选）
   - 设置"如果任务已经运行，以下规则适用"为"请勿启动新实例"

5. **完成**
   - 点击"完成"
   - 任务已创建

#### 立即测试任务

1. 在任务计划程序中找到 `DestinyTeller-AutoScheduler`
2. 右键 → "运行"
3. 打开任务管理器，查看是否有 `python.exe` 进程
4. 查看日志：`logs/scheduler/2026-02-06_daily_scheduler.log`

## 常见问题

### Q: Cookie 过期怎么办？

**A:** 重新运行发布测试：
```bash
python test_publish_manual.py
```
手动登录后，Cookie 会自动更新。

### Q: 如何停止调度器？

**A:**
- **临时运行**：关闭命令行窗口
- **开机启动**：任务计划程序 → 找到任务 → 右键"禁用"或"删除"
- **强制停止**：任务管理器 → 结束 python.exe 进程

### Q: 如何查看发布是否成功？

**A:**
1. 查看日志文件
2. 查看 `content/published/日期/` 目录
3. 检查小红书账号的"我的笔记"

### Q: 可以修改发布时间吗？

**A:** 可以，编辑 `automation/config/schedule.json`，修改 `optimal_time` 字段。

### Q: 可以修改内容风格吗？

**A:** 可以，编辑 `automation/generators/claude_content_generator.py` 中的示例内容。
