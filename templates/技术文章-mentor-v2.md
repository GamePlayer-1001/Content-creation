# 三行注释救了我的命——一个关于项目文档的顿悟

> **平台**: LinuxDo
> **版本**: C+G V2 (人性化优化版)
> **风格**: Mentor (导师型散文叙事)
> **生成时间**: 2025-12-22
> **优化说明**: 针对腾讯混元检测结果优化，打破对仗句式，增加口语碎碎念

---

那天凌晨两点，我把测试环境改崩了。

屏幕上一片红。我盯着报错，脑子里一片空白。这个项目接手才一个月，十八个文件夹，两百多个文件。README写得像诗一样漂亮。一个字都没更新过。

我删了一个工具函数。八百里外有人在用它。

我当时坐在那里，突然想起一件事。

---

大概十年前，我读《哥德尔、埃舍尔、巴赫》。里面有个概念叫自指。「这句话是假的」——你想证明它是真的，它就变成假的；你想证明它是假的，它又变成真的。一个永远无法稳定的怪圈。

当时觉得是哲学家的玩具。没想到会在凌晨两点想起来。

我盯着满屏的报错，脑子里冒出一个念头：**文档不会自己更新，是因为「更新文档」和「写代码」是两件事。**

你想啊，代码你天天在改，文档呢？上次更新是什么时候？上个月？上个季度？反正不是今天。

那能不能让它俩变成同一件事？

---

我试了个特别蠢的办法。

就是在每个文件开头写三行注释。就三行。

```javascript
/**
 * INPUT: 依赖 config.js 的数据库配置
 * OUTPUT: 提供用户认证中间件
 * POS: 核心中间件，被所有API路由依赖
 */
```

INPUT写这文件吃什么进去。OUTPUT写它吐什么出来。POS写它在整个项目里算老几。

第一次写的时候我觉得挺傻的，像小学生做填空题。

但是一个月后我回来改这文件。瞥一眼开头，哦，它依赖config，被路由层调用。那我改了它，config那边不用动，但路由层得测一遍。

就这么简单。但真的有用。

---

后来我又往前走了一步。我给每个文件夹加了个 `INDEX.md`。

你别看不起这东西。就十来行。

```markdown
# /services
业务逻辑层。不碰数据库，只调 /models。

- `userService.js` - 用户那套
- `authService.js` - 登录认证
- `emailService.js` - 发邮件的

---
动了这里面的东西？记得回来改我。
```

然后根目录也搞一个README，把所有文件夹串起来。像树根一样。

关键的地方来了。我在每个文件最后都加了一句话：**你要是改了我，记得顺手把文件头和INDEX也改了。**

这就形成了一个套娃。改文件要改头。改头要改INDEX。改INDEX要改根目录。

一环扣一环，跑不掉。

---

三个月我们招了个新人。

他第一天来，我说你看根目录README，然后顺着往下找。他十五分钟就开始写代码了。

以前？两天。还得有人带。

想删个文件，先看它OUTPUT被谁用着。想改个接口，看INPUT知道会波及谁。不用猜，不用全局搜，低头就能看见。

最爽的是code review的时候。有人改了代码没改文件头，直接打回去。不是我卡你，是流程卡你。

---

这事儿为什么能成？我琢磨过。

第一，它够简单。三行注释十秒钟写完。你让我写个两百字的说明文档我肯定懒得写，但三行？行吧。

第二，它是强制的。不是靠自觉。我们后来加了个git hook，你要是改了文件没改对应的INDEX，commit的时候会弹个警告问你确定吗。大部分人看到警告就老实了。

第三，它是分形的。这词听着唬人，其实就是每一层都长一样。文件头三行，文件夹十行，根目录五十行。格式都一样，脑子不用切换。

---

模板直接给你们，复制就能用。

Python的：
```python
"""
INPUT: user_model.User, config.DATABASE_URL
OUTPUT: get_user(id), create_user(data)
POS: 数据访问层-用户表
"""
```

JS的：
```javascript
/**
 * INPUT: axios, ../config/api.js
 * OUTPUT: fetchUserData(), updateUserProfile()
 * POS: API调用层-用户相关
 */
```

文件夹的INDEX.md就照着上面那个抄。Git hook网上一搜一大把，我就不贴了。

---

有几个坑说一下。

INDEX别写太长。三五行足够了。你写成论文，下次谁来维护？没人。

格式别太死。有人爱写「依赖」有人爱写「需要」，无所谓，能看懂就行。

还有，别一上来就全项目铺开。先从最核心的三个模块开始搞。有效果了再说。

---

这套东西我也不知道该叫什么名字。分形索引？多级目录？随便吧。

反正就三件事：文件头三行，文件夹一个INDEX，根目录一个README。没了。

那天凌晨两点的崩溃，现在想想，好像也不全是坏事。

你要是也被「文档永远是旧的」这事折磨过，试试看。
